Getting Started On Biowulf

See Biowulf User Guide for more information.

ExomePipeline was developed to work on the NIH Biowulf cluster. So, this will require that you have an account with user name and password for Helix and Biowulf2. Additionally, you will need to run your favorite SSH and telnet client software such as Terminal for Mac OS X or PuTTY for Windows. This example is given from Macintosh's Terminal App.

1. SSH connection to the Biowulf Cluster and launching CCBR Pipeliner (Return to Main Page)

ssh - Creates a connection with Biowulf.
Launch Macintosh's Terminal application from the Utility directory and type in the following lines

# This is the pretend terminal window
% ssh -Y @biowulf2
% password:
biowulf %
biowulf % module load ccbr
biowulf % module load pipeliner
biowulf % runpipe


CCBR Pipeliner will launch giving you the following graphical user interface below.

2. Project Information Tab (Return to Main Page)

Select the Project Information tab and fill it in with the appropriate information.

3. Pipeline Options Tab (Return to Main Page)

Select the Pipeline Options tab.

Initial File Type: Select the input data format. In this example, we will start with fastq.gz files. We suggest conversion and storage of fastq files as fastq.gz files with as much as a 75% reduction in file size.

Software Set: Currently, Standard-bin is the only option.

Annotation Set: Currently, hg19 and mm10 are currently being supported.


Pipeline: Select initalqc for this example. CCBR provides the following pipelines:
  • initialqc - This pipeline runs a set of Quality Control software on your data.
  • bam2recal -
  • exomeseq-somatic -
  • exomeseq-germline -
  • exomeseq-germline-recal -
  • exomeseq-germline-partial -
  • custom -

    • Final File Type: Select fastq for this example.

    Full Path to Data Directory: For example /home/user_name/projects/ccbr_xxxx/Data
    This needs to be an absolute path to your data directory. In this case, a set of fastq.gz files.

    Full Path to Working Directory: Create a new empty directory such as /home/user_name/projects/ccbr_xxxx/Exome_germline_analysis.
    WARNING: We will "Initialize" this working directory later on which will delete everything inside. This directory should be different from the Data directory. You may copy and paste the paths to the two directs or use the File Menu.


    4. Project Json Tab (Return to Main Page)

    Select the Project Json Tab. This will allow you to review the JSON file which contains the project information. You can select any of the prior tabs discuss to adjust Project settings contained in the JSON file.

    5. Run Sequence Tab (Return to Main Page)

    Select the Run Sequence Tab. This will walk you though the sequence running up to the submitting the run to Biowulf.

    Initialize Working Directory: In this step a directory, which must already exist, will be initialized for Pipeliner. Initializatin involves deleting all preexisting directory content and writing a few subdirectories containing a colletion of data files and scripts to be used locally diring the pipeline run.

    Symbolic Link Data: The starting data file will be linked from the Data Directory to the Working Directory.

    Preform Dry Run: A test will be preformed to verify that the pipeline will run as expected. No data is processed at this point.

    Save Project Profile: The pipeline parameters, project information and comments are saved.

    Submit Job to Biowulf2 via Slurm: The pipeline job is submitted to the batch queuing system on the Biowulf2 cluster.
